# Accurat System — Комплексная система управления автомойкой

## Описание проекта

**Accurat** — это клиент-серверный программный комплекс для автоматизации бизнес-процессов сети автомоек. Проект построен по принципу монорепозитория (monorepo) и включает в себя:

* Производительный десктопный терминал администратора (WPF)
* Мобильное приложение для оперативного контроля (MAUI)
* Масштабируемый REST API сервер с централизованной базой данных PostgreSQL.

Система позволяет управлять несколькими филиалами, вести сложный финансовый учет, контролировать кассовые смены, управлять расписанием боксов и автоматически рассчитывать заработную плату сотрудников с поддержкой гибкого распределения ролей (Soft Split).

## Структура репозитория

Проект разделен на три основные директории:

* **`AccuratPanelCarWashing/`** — Десктопный клиент. Основное рабочее место администратора, разработанное на WPF (паттерн MVVM). Предназначено для создания заказов, ведения кассы и генерации отчетов в Excel.
* **`AccuratPanelCWM/`** — Мобильный клиент. Кроссплатформенное приложение на .NET MAUI. Дублирует ключевой функционал десктопной версии (просмотр и закрытие активных заказов) с адаптивным интерфейсом и поддержкой системных тем (Светлая/Темная).
* **`Akkurat.WebAPI/`** — Серверная часть. REST API бэкенд на ASP.NET Core, инкапсулирующий всю бизнес-логику, вычисления, математику зарплат и взаимодействие с базой данных PostgreSQL.

## Основные возможности

### Архитектура и Учет заказов

* **Мультифилиальность:** Управление всей сетью моек из единой базы данных со строгим разделением прав и доступа по `BranchId`.
* **Real-time синхронизация:** Использование **SignalR** для мгновенного обновления интерфейсов WPF и MAUI при изменении данных на сервере.
* **Единый конвейер заказов:** Предварительные записи и фактические заказы реализованы как единая сущность в БД с динамической системой статусов.
* **Паттерн Outbox:** Асинхронная обработка событий (например, при завершении заказа) без блокировки основного потока API. Фоновый воркер `OutboxProcessorBackgroundService` гарантирует доставку и обработку системных сообщений.

### Финансовый модуль, Зарплаты и Касса

* **Soft Split (Умное распределение ЗП):** Связь "многие-ко-многим" (`OrderWashers`) позволяет назначать на один заказ нескольких мойщиков, гибко разделяя между ними выручку (долю участия).
* **Кастомные ставки:** Математическое ядро на сервере поддерживает гибкую настройку ЗП: базовый процент для каждого отдельного сотрудника (например, 35% для стажера, 40% для профи) и переопределяющий (индивидуальный) процент для конкретной услуги (например, 15% за нанесение керамики).
* **Контроль потоков:** Учет фактического остатка в кассе с разделением по типам оплат (Наличные, Карта, Перевод, QR-код). Раздельный учет расходов на нужды мойки и инкассаций.
* **Авансы:** Именная выдача авансов сотрудникам с автоматическим вычетом из итоговой зарплаты при закрытии смены.

### Расписание и CRM

* **Умное расписание:** API-проверка доступности моечных боксов и сервисных подъемников с блокировкой пересечений времени при записи.
* **Клиентская база:** Сохранение истории визитов, расчет LTV (Total Spent) и счетчик посещений. Глобальный поиск и автозаполнение данных при вводе государственного номера автомобиля.

### Отчетность и Экспорт

* Генерация интервальных финансовых отчетов за любой выбранный период с разделением по филиалам.
* Экспорт аналитики, кассовых смен и зарплатных ведомостей в форматы Excel (через ClosedXML) и JSON из десктопного приложения.

## Технологический стек

**Backend (API):**

* C# 13, ASP.NET Core (.NET 10)
* PostgreSQL 12+ (Entity Framework Core)
* SignalR (WebSockets)
* Swagger / Scalar (документация API)

**Frontend (Desktop - WPF):**

* C# 7.3, WPF (.NET Framework 4.8)
* ClosedXML (работа с Excel)
* Newtonsoft.Json
* SignalR Client

**Frontend (Mobile - MAUI):**

* C# 13, .NET MAUI (.NET 10)
* MVVM (собственная реализация)
* `Microsoft.Maui.Storage.Preferences` для локального хранения настроек

## Требования к окружению и Запуск

### 1. Подготовка базы данных и сервера

1. Убедитесь, что установлен .NET 10 SDK и сервер PostgreSQL.
2. Перейдите в папку бэкенда:
```bash
cd Akkurat.WebAPI

```


3. Откройте `appsettings.json` и настройте строку подключения `DefaultConnection` к вашей БД PostgreSQL.
4. Примените миграции Entity Framework для создания таблиц:
```bash
dotnet ef database update

```


5. Запуск сервера:
```bash
dotnet run

```



### 2. Запуск клиентских приложений

1. Откройте общее решение `AccuratSystem.sln` или `AccuratSystem.slnx` в Visual Studio 2022.
2. **Для WPF:** Убедитесь, что в классе `ApiService` указан корректный базовый URL запущенного Web API сервера (например, локальный localhost или туннель). Установите `AccuratPanelCarWashing` как запускаемый проект и нажмите "Пуск".
3. **Для MAUI:** Выберите проект `AccuratPanelCWM`, настройте целевое устройство (Android Emulator или локальное Windows-приложение) и нажмите "Пуск". Настройки подключения к серверу можно менять прямо из интерфейса мобильного приложения на вкладке "Настройки".
