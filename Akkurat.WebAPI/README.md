# Accurat Web API

Backend часть программного комплекса для управления сетью автомоек "Accurat". Обеспечивает централизованное хранение данных, бизнес-логику и взаимодействие с клиентским WPF-приложением через REST API.

## Описание

Данный проект представляет собой серверное приложение на базе ASP.NET Core, спроектированное для обслуживания сети автомоек. Сервер берет на себя всю тяжелую бизнес-логику: от вычисления сложных перекрестных зарплатных ведомостей до проверки доступности моечных боксов при онлайн-записи, разгружая тем самым клиентские приложения.

## Основные возможности

*   **Единая архитектура данных:** Использование PostgreSQL для надежного хранения истории заказов, клиентов и финансов всех филиалов сети.
*   **Мультифилиальность:** Строгое разделение данных (смены, транзакции, заказы, сотрудники) на уровне запросов к базе данных по идентификатору филиала (BranchId).
*   **Сложная финансовая аналитика:** 
    *   Обработка кассовых транзакций (приходы, расходы, инкассации).
    *   Автоматический расчет кассовых смен.
    *   Формирование зарплатных ведомостей с учетом процентной ставки, выданных авансов и гарантированной минимальной оплаты за выход.
*   **Управление нагрузкой:** Алгоритмы проверки занятости моечных боксов в реальном времени для предотвращения накладок при записи.
*   **Унифицированные сущности:** Предварительные записи и фактические заказы объединены в единую таблицу базы данных для исключения дублирования бизнес-логики и конфликтов при конвертации.

## Технологический стек

*   **Фреймворк:** ASP.NET Core (.NET 10)
*   **База данных:** PostgreSQL
*   **ORM:** Entity Framework Core
*   **Документация API:** Swagger / Scalar

## Требования к окружению

*   .NET 10 SDK
*   СУБД PostgreSQL (версии 12 или новее)

## Развертывание и запуск

1. Клонирование репозитория:
   ```bash
   git clone [https://github.com/iamfifya/Akkurat.WebAPI.git](https://github.com/iamfifya/Akkurat.WebAPI.git)
   cd Akkurat.WebAPI

2. Настройка базы данных:
Откройте файл appsettings.json (или appsettings.Development.json) и укажите корректные параметры подключения к вашему серверу PostgreSQL в секции ConnectionStrings.

3. Применение миграций (создание структуры БД):

Bash
   dotnet ef database update
4. Запуск сервера:

Bash
dotnet run
Сервер будет доступен по адресу, указанному в конфигурации (по умолчанию https://localhost:7165).

Структура проекта
Controllers/ — обработчики REST API запросов для клиентов, заказов, финансов и смен.

Models/ — сущности базы данных и DTO (Data Transfer Objects) для передачи сводных отчетов.

Data/ — контекст базы данных (AppDbContext) и конфигурация Entity Framework.
